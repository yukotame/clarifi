# ClariFi 要件定義書 v1.0

最終更新: 2026-05-17

---

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| アプリ名 | ClariFi（クラリファイ） |
| アプリ種別 | Webアプリ（2アプリ構成） |
| 利用者 | 自分のみ（将来的にポートフォリオとして公開） |
| 目標 | 最初はMVP（最小構成）→ 将来的にポートフォリオ公開 |
| データ保存場所 | 自PC・ローカルサーバーのみ（外部クラウドなし） |

---

## 2. アプリ構成

```
【アプリ①】 WAON明細 自動取得 → CSV出力アプリ
【アプリ②】 WAON CSV + PayPay CSV → 支出 見える化アプリ
```

---

## 3. アプリ① WAON CSV出力アプリ

### 3-1. 機能要件

| # | 機能 | 詳細 |
|---|------|------|
| F1-1 | WAON公式サイト 自動ログイン | ログインID・パスワードをアプリ内に保存し自動ログイン |
| F1-2 | 明細データ 自動取得 | 公式サイトから直近1か月分の明細をスクレイピングで取得 |
| F1-3 | CSV出力 | 取得した明細を指定フォーマットでCSV出力 |
| F1-4 | 自動実行 | node-cronで定期的に自動取得・出力を実行 |

### 3-2. CSV出力フォーマット

| カラム名 | 内容 | 例 |
|----------|------|----|
| 取引年月日 | 取引日付 | 2026-05-01 |
| 利用店舗 | 店舗名 | イオン○○店 |
| 利用金額（税込） | 金額（円） | 1980 |
| 利用区分 | 利用の種別 | お買い物 |
| チャージ区分 | チャージ種別 | チャージ / 空白 |

### 3-3. 非機能要件

- WAONカードは1枚
- ログイン情報は `.env` に保存（Gitには含めない）
- 対象期間：直近1か月

## 4. アプリ② 支出見える化アプリ

### 4-1. 機能要件

| # | 機能 | 詳細 |
|---|------|------|
| F2-1 | WAON CSV インポート | アプリ①が出力したCSVを読み込む |
| F2-2 | PayPay CSV インポート | PayPay公式アプリから手動エクスポートしたCSVを読み込む |
| F2-3 | データ正規化 | 2つのCSVを統一フォーマットに変換して統合管理 |
| F2-4 | 支出フィルタリング | チャージ・返金・ポイント利用を自動除外 |
| F2-5 | カテゴリ自動分類 | 店舗名をもとに自動分類 |
| F2-6 | グラフ表示① | 月次支出合計（棒グラフ） |
| F2-7 | グラフ表示② | カテゴリ別内訳（円グラフ） |

### 4-2. カテゴリ定義（初期）

| カテゴリ名 |
|------------|
| 食費 |
| 日用品 |
| 外食 |
| 医療費 |
| 交通費 |
| 娯楽 |
| 子ども関連 |
| その他 |

### 4-3. 除外ルール

| 対象 | 処理 |
|------|------|
| チャージ取引 | 支出集計から除外 |
| 返金 | 支出集計から除外 |
| ポイント利用 | 支出集計から除外 |

### 4-4. 画面構成（初期版）

| 画面 | 内容 |
|------|------|
| ダッシュボード | 月次棒グラフ＋カテゴリ円グラフ |
| CSVインポート画面 | WAON / PayPay CSVの読み込み |

### 4-5. 非機能要件

- PCブラウザ最適化
- 重複チェックなし（初期版）
- データ修正機能なし（初期版）

## 6. 制約・注意事項

- WAONの自動取得は**個人利用・非公開の範囲**で使用すること
- `.env` は絶対にGitにコミットしない
- WAONサイト構造変更によりスクレイピングが壊れる可能性あり
